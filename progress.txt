### What was done
1. **`app/scraping/base_scraper.py`** — BaseScraper class:
   - Rate limiting via `asyncio.Semaphore` (configurable `SCRAPER_CONCURRENCY`, default 3)
   - Retry with exponential backoff (`2**attempt` seconds, max `SCRAPER_RETRY_MAX` = 3)
   - Exception hierarchy: `ScraperError` → `ScraperTimeoutError`, `ScraperHTTPError`, `ScraperParseError`
   - Retryable check: retries on timeouts, connect errors, HTTP 5xx
   - Structured logging: URL, status code, elapsed time, attempt count
   - `httpx.AsyncClient` with lazy init, async context-manager support
2. **`app/scraping/fifa_scraper.py`** — FIFAScraper(BaseScraper):
   - `scrape_match_schedule()` → `ScrapedSchedule`: extracts `__NEXT_DATA__` JSON from FIFA schedule page
   - `scrape_match_result(match_id)` → `ScrapedMatchResult`: extracts match result with events
   - Multiple JSON path fallbacks for FIFA page structure variations
   - Graceful degradation: returns empty data when parsing fails
   - Configurable URLs from `settings.FIFA_SCHEDULE_URL` / `settings.FIFA_MATCH_URL`
3. **`app/schemas/scraper_schema.py`** — Pydantic validation models:
   - `ScrapedMatch`, `ScrapedSchedule`, `ScrapedEvent`, `ScrapedMatchResult`
   - Field validators for stage, status, team_side
4. **`app/config.py`** — Added scraper settings:
   - `FIFA_SCHEDULE_URL`, `FIFA_MATCH_URL`, `SCRAPER_CONCURRENCY`, `SCRAPER_TIMEOUT`, `SCRAPER_RETRY_MAX`
5. **`.env.example`** — Updated with new scraper config entries
